# PRD：账单确认 - 已支付 Tab

## 1. 概述

- **需求名称**：账单确认新增「已支付」Tab
- **需求背景**：当前账单确认页面仅有「待处理」和「已处理」两个 Tab，用户无法快速查看已完成支付的订单记录。需要新增「已支付」Tab，方便用户查询和核对已付款订单。
- **目标用户**：使用账单确认小程序的财务/采购人员
- **业务目标**：让用户能便捷地查看已支付订单，支持按订单号或业务方快速检索，提升对账效率

## 2. 问题陈述

- **当前痛点**：已支付的订单没有独立查看入口，用户无法快速找到某笔已付款订单的支付详情
- **现有方案不足**：「已处理」Tab 可能混合了多种状态的订单，不能直接聚焦已支付记录
- **不解决会怎样**：用户需要在其他系统或线下核对支付记录，效率低、容易出错

## 3. 解决方案

- **方案概述**：在账单确认页面新增「已支付」Tab，展示所有已完成支付的订单列表，支持搜索和展开查看详情
- **核心功能点**：
  1. 新增「已支付」Tab 页签
  2. 已支付订单列表（收起态仅展示订单号 + 业务方）
  3. 搜索功能（按订单号或业务方检索）
  4. 订单详情展开（展示完整支付信息）
- **方案优势**：操作路径短、信息层级清晰、支持快速定位目标订单

## 4. 用户故事

### US-1：查看已支付订单列表
> 作为财务人员，我想要在账单确认页面查看已支付的订单列表，以便快速了解哪些订单已完成付款

**验收标准（AC）：**
- 账单确认页面 Tab 栏新增「已支付」页签，位于「已处理」右侧
- 进入已支付 Tab 后展示订单列表
- 列表每条记录收起态展示：订单号、业务方
- 列表按支付时间倒序排列（最近支付的在最上面）

### US-2：搜索已支付订单
> 作为财务人员，我想要通过订单号或业务方搜索已支付订单，以便快速定位目标订单

**验收标准（AC）：**
- 列表上方有搜索输入框，placeholder 为「搜索订单号或业务方」
- 输入关键词后实时过滤列表
- 搜索无结果时展示空状态提示

### US-3：查看已支付订单详情
> 作为财务人员，我想要展开某条已支付订单查看完整支付信息，以便核对账单明细

**验收标准（AC）：**
- 点击列表项展开详情区域，再次点击收起
- 详情展示以下字段：业务方（订货/后台导入）、收款方、下单时间、支付时间、支付金额
- 同一时间可展开多条记录

## 5. 功能详细设计

### 5.1 Tab 页签

- **功能描述**：在现有 Tab 栏末尾新增「已支付」页签
- **实现规则**：
  - Tab 顺序：待处理 → 已处理 → 已支付
  - 「已支付」Tab 不显示角标数字（与待处理不同）
  - 切换 Tab 时保留搜索框内容不清空
- **交互说明**：点击 Tab 切换，选中态有底部指示条（与现有样式一致）

### 5.2 订单列表

- **功能描述**：展示已支付订单的卡片列表
- **实现规则**：
  - 收起态展示：订单号（左上主信息）、业务方标签（左下）、支付金额（右侧）
  - 业务方取值：「订货」或「后台导入」
  - 默认按支付时间倒序排列
  - 列表支持下拉刷新和上拉加载更多
- **交互说明**：
  - 点击卡片任意区域展开/收起详情
  - 展开时右侧箭头旋转 90°
- **异常处理**：
  - 无数据时展示空状态：图标 + 「暂无已支付订单」

### 5.3 搜索功能

- **功能描述**：通过关键词过滤已支付订单列表
- **实现规则**：
  - 搜索范围：订单号（模糊匹配）、业务方（精确匹配「订货」或「后台导入」）
  - 输入即搜索，无需点击搜索按钮
  - 搜索结果实时更新列表和汇总数据（笔数、合计金额）
- **交互说明**：
  - 输入框获得焦点时键盘弹起
  - 输入框右侧有清除按钮（有内容时显示）
- **异常处理**：
  - 搜索无结果展示空状态提示

### 5.4 订单详情（展开态）

- **功能描述**：展开卡片显示完整支付信息
- **实现规则**：
  - 展示字段及顺序：
    1. 业务方：订货 / 后台导入
    2. 收款方：具体收款方名称
    3. 下单时间：YYYY-MM-DD HH:mm 格式
    4. 支付时间：YYYY-MM-DD HH:mm 格式
    5. 支付金额：¥XX,XXX.XX 格式（千分位）
  - 详情区域与列表摘要之间有分割线
- **交互说明**：展开/收起有过渡动画（0.25s ease）

## 6. 边界范围

### IN SCOPE（本期做）
- 新增「已支付」Tab 页签
- 已支付订单列表展示（订单号 + 业务方）
- 搜索过滤（订单号、业务方）
- 展开查看订单详情

### OUT OF SCOPE（本期不做）
- 订单状态修改/撤销支付
- 导出已支付订单列表
- 已支付订单的批量操作
- 按时间范围筛选
- 权限控制

### 依赖项
- 后端提供已支付订单列表接口（支持分页、搜索）

## 7. 非功能需求

- **性能**：列表首屏加载 ≤ 1s，搜索响应 ≤ 300ms
- **兼容性**：与现有小程序环境一致
- **体验**：交互风格与现有「待处理」「已处理」Tab 保持一致
